## Diagnosis Summary (before any remediation)

### Candidate source clusters considered
1. Spec/template drift between BRD schema and runtime validators.
2. Regex-based parsing of markdown sections (state/alias/report parsing fragility).
3. Command gate criteria implemented with surrogate signals (not true BRD gate source).
4. Platform-specific safety gaps (Windows path semantics and reserved names).
5. Compile target drift (target matrix vs actual dispatcher support).
6. Budget constants interpreted as max table instead of default table.
7. Safety sanitizer incompleteness for shell metacharacter families.

### Most likely root causes
- **RC-1:** Schema/contract drift across generators vs validators (templates, check, config, budgets, compile targets).
- **RC-2:** String/regex heuristics used where structured parsing is required (DEV-ENV extraction, migration report parsing, shell/path safety).

## Bug Findings

### B-001
- **Severity:** HIGH
- **Component:** `src/commands/check.ts`
- **What happens:** `check.execute()` flags generated `stackmoss.config.json` as invalid (`projectName` missing), even when file is generated by the official template.
- **What should happen:** Generated config should pass sanity checks without schema false positives.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-001: check should not report structure_invalid on freshly generated stackmoss.config.json"**.

### B-002
- **Severity:** HIGH
- **Component:** `src/commands/check.ts`
- **What happens:** Regex `\[CAP\]([\s\S]*?)(?=\n\[|$)` over-captures beyond the capability into unrelated sections (for example `PROJECT_FACTS`), causing false `budget_exceeded`.
- **What should happen:** Budget count must be scoped to the capability block only.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-002: check should not over-count capability budget from unrelated sections"**.

### B-003
- **Severity:** HIGH
- **Component:** `src/commands/run.ts`
- **What happens:** `resolveAlias` fails to parse a valid `[DEV-ENV]` alias from `team.md`; the regex uses a malformed escape and misses the alias.
- **What should happen:** `stackmoss run <alias>` must resolve aliases from `[DEV-ENV]` when package scripts are absent.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-003: run alias resolution should parse [DEV-ENV] aliases in team.md"**.

### B-004
- **Severity:** CRITICAL
- **Component:** `src/commands/promote.ts`
- **What happens:** Promote treats `package.json` script existence as `[DEV-ENV] verified`, allowing promote without true DEV-ENV verification source.
- **What should happen:** Criterion #2 must verify at least one `[DEV-ENV]` command per BRD gate semantics.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-004: promote must fail criterion #2 if no [DEV-ENV] command was verified"**.

### B-005
- **Severity:** HIGH
- **Component:** `src/intake/pack-selector.ts`
- **What happens:** Missing `Q_PT` silently defaults to `MVP`.
- **What should happen:** Critical decision fields should not silently default; they should fail or remain unresolved.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-005: missing Q_PT should be treated as unresolved critical input (no silent default)"**.

### B-006
- **Severity:** MEDIUM
- **Component:** `src/templates/team.ts`, `src/budgets.ts`
- **What happens:** Generated capability budgets use max values (for example `TL-ARCH = 280`) instead of the BRD default table (`TL-ARCH = 220`).
- **What should happen:** Initial generated `team.md` should emit default budget values.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-006: team template should emit BRD default budget for TL-ARCH (220), not inflated max budget"**.

### B-007
- **Severity:** MEDIUM
- **Component:** `src/compile/index.ts`
- **What happens:** BRD target map includes Roo role-level target but dispatcher rejects `Roo` as unsupported.
- **What should happen:** Roo target should be supported or BRD/contract must be explicitly version-gated and aligned.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-007: compile target should support Roo role-level output per BRD target map"**.

### B-008
- **Severity:** HIGH
- **Component:** `src/scanner/report.ts`
- **What happens:** Parser counts unresolved `- [ ] **[` anywhere in the file, not only inside the `OPEN QUESTIONS` section; this can block promote on unrelated checklist text.
- **What should happen:** Only the `OPEN QUESTIONS` section should contribute unresolved counts.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-008: parser should count unresolved items only inside OPEN QUESTIONS section"**.

### B-009
- **Severity:** HIGH
- **Component:** `src/commands/run.ts`
- **What happens:** `isAliasSafe` allows newline and redirection payload styles (`\n`, `>`) which can alter command behavior.
- **What should happen:** Sanitizer should block broader metacharacter/injection vectors, including redirection and newline chaining.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-009: alias safety should reject newline/redirection command injection patterns"**.

### B-010
- **Severity:** CRITICAL
- **Component:** `src/patch/engine.ts`
- **What happens:** Absolute Windows paths outside project root can bypass traversal check and be modified.
- **What should happen:** Any target outside project root must be rejected.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-010: patch apply should block absolute path escape outside project root"**.

### B-011
- **Severity:** MEDIUM
- **Component:** `src/commands/new.ts`
- **What happens:** Reserved Windows device names (`CON`, `NUL`, `PRN`) are accepted by `new` arg parser.
- **What should happen:** Reserved names should be blocked to avoid filesystem/runtime corruption.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-011: new command should reject dangerous Windows reserved names"**.

### B-012
- **Severity:** MEDIUM
- **Component:** `src/compile/cursor.ts`
- **What happens:** `constitution`, `project-facts`, and `working-contract` outputs are placeholders, not extracted content.
- **What should happen:** Always-on rules should contain actual authoritative content sections.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-012: Cursor always-on rules should contain real extracted content, not placeholders"**.

### B-013
- **Severity:** LOW
- **Component:** `src/templates/config.ts`
- **What happens:** Generated config defaults to `ClaudeCodeV2` target naming, diverging from BRD base target nomenclature (`ClaudeCode`).
- **What should happen:** Target naming should align with the BRD contract or be explicitly documented as a versioned override.
- **Proof:** [`tests/adversarial/stretch-brd-mismatches.test.ts`](tests/adversarial/stretch-brd-mismatches.test.ts) - **"B-013: generated config target should match BRD base target name ClaudeCode"**.

## Round 2 Findings

Merged below are both round-2 batches. Per request, the current
[`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts)
batch remains `B-014..B-020`, and the second batch is renumbered to
`B-021..B-027`. Some items are closely related in surface area, but none are
discarded.

### B-014
- **Severity:** HIGH
- **Component:** `src/commands/new.ts`
- **Actual:** `checkState()` and `execute()` are not atomic. If the target folder appears after `checkState()` passes, `execute()` still succeeds due to `mkdirSync(..., { recursive: true })`.
- **Expected:** Existing folder must abort clearly and never be silently reused/overwritten (BRD §16 acceptance #11, atomic safety expectations in BRD §16 #10).
- **Failing test:** [`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts) - **"B-014: new command should abort if target folder appears between checkState and execute (race window)"**.

### B-015
- **Severity:** HIGH
- **Component:** `src/intake/interview-mode.ts`
- **Actual:** Interview completeness gate only checks `Q2` and `Q12`; it never enforces the required "at least 1 non-goal" condition, so no targeted follow-up is triggered for that gap.
- **Expected:** Completeness gate must enforce all 3 conditions: for-who, measurable outcome, and at least one non-goal; if missing, ask targeted follow-ups with max 2 (BRD §8.3).
- **Failing test:** [`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts) - **"B-015: interview completeness gate should request follow-up when non-goal is missing"**.

### B-016
- **Severity:** MEDIUM
- **Component:** `src/templates/team.ts`
- **Actual:** `QA(light)` role variant is rendered with full QA capabilities including `[QA-REGRESSION]`, effectively behaving like strong QA.
- **Expected:** `QA(light)` and `QA(strong)` must remain distinct per pack matrix semantics (BRD §10.2 role mapping table).
- **Failing test:** [`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts) - **"B-016: team template should honor QA(light) variant and exclude QA-REGRESSION capability"**.

### B-017
- **Severity:** MEDIUM
- **Component:** `src/compile/antigravity.ts`
- **Actual:** Compiling `QA(light)` emits `quality-assurance--regression/SKILL.md`, collapsing light-vs-strong distinction at compile time.
- **Expected:** Compile layer must preserve matrix-driven role variants instead of upgrading light QA into strong QA capabilities (BRD §10.2, BRD §15 compile consistency intent).
- **Failing test:** [`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts) - **"B-017: antigravity compile should not emit regression skill for QA(light)"**.

### B-018
- **Severity:** HIGH
- **Component:** `src/templates/features.ts`
- **Actual:** Intake free-text `firstFeature.name` is injected directly into markdown heading context; malicious markdown headers create extra sections in generated `FEATURES.md`.
- **Expected:** Generated file schema should remain stable and not be breakable by intake text payloads (BRD §9.3 fixed schema, BRD §3 safe-by-default principle).
- **Failing test:** [`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts) - **"B-018: feature template should prevent markdown header injection from intake answer"**.

### B-019
- **Severity:** HIGH
- **Component:** `src/commands/promote.ts`
- **Actual:** Promote gate evaluates unresolved questions only from `MIGRATION_REPORT.md`; unresolved `OPEN_QUESTIONS.md` can be ignored and promotion still succeeds.
- **Expected:** Unresolved open questions lifecycle must block promote until cleared (BRD Appendix B: `OPEN_QUESTIONS.md` lifecycle + promote gate behavior).
- **Failing test:** [`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts) - **"B-019: promote should block when OPEN_QUESTIONS.md still has unresolved items"**.

### B-020
- **Severity:** CRITICAL
- **Component:** `src/commands/upgrade.ts`
- **Actual:** Constitution replacement uses `indexOf('## CONSTITUTION')` without header anchoring; hostile `team.md` containing that token inside `PROJECT_FACTS` can cause mutation of facts during upgrade.
- **Expected:** Upgrade must never overwrite or mutate `PROJECT_FACTS` under any content shape (BRD §14 HARD RULE).
- **Failing test:** [`tests/adversarial/stretch-round2.test.ts`](tests/adversarial/stretch-round2.test.ts) - **"B-020: upgrade constitution replace should not mutate PROJECT_FACTS when team.md is hostile"**.

### B-021
- **Severity:** HIGH
- **Component:** `src/patch/engine.ts`
- **Actual:** `applyProposal()` ignores `suggestedFix.section` and replaces the first matching `oldContent` anywhere in the file, so a DEV-ENV patch can mutate `CONSTITUTION` instead of the intended section.
- **Expected:** Patch apply must find the declared section/capability and rewrite that section only per BRD §12.3 ("Tim section/capability chua thong tin cu -> rewrite section do").
- **Failing test name:** **"B-014: patch apply must update only the declared section, not the first duplicate match in the file"**.

### B-022
- **Severity:** HIGH
- **Component:** `src/patch/engine.ts`
- **Actual:** Patch apply only enforces `newContent <= oldContent` word count, but never re-checks the BRD hard max for the target capability or team total, so an oversized capability can still be written.
- **Expected:** Patch apply must enforce the budget gate and trim/reject content that still exceeds BRD §12.3 and BRD §12.4 limits.
- **Failing test name:** **"B-015: patch apply should enforce BRD max budget after mutation, not only new<=old word count"**.

### B-023
- **Severity:** HIGH
- **Component:** `src/commands/promote.ts`
- **Actual:** `promote` only reads `MIGRATION_REPORT.md`; unresolved items in `OPEN_QUESTIONS.md` are ignored, so promotion can succeed with a ghost unresolved file still present.
- **Expected:** Promote must block when `OPEN_QUESTIONS.md` exists with unresolved items per BRD Appendix B lifecycle and BRD §13.2 gate semantics.
- **Failing test name:** **"B-016: promote should block when OPEN_QUESTIONS.md still has unresolved items"**.

### B-024
- **Severity:** CRITICAL
- **Component:** `src/commands/upgrade.ts`
- **Actual:** `applyUpgrade()` locates `## CONSTITUTION` with naive `indexOf`, so a hostile `## CONSTITUTION` marker inside `PROJECT_FACTS` is treated as the upgrade target and gets rewritten.
- **Expected:** Upgrade must never touch `PROJECT_FACTS` under any circumstance per BRD §14 hard rule.
- **Failing test name:** **"B-017: upgrade must not rewrite PROJECT_FACTS when a hostile marker contains ## CONSTITUTION text"**.

### B-025
- **Severity:** HIGH
- **Component:** `src/scanner/report.ts`, `src/commands/promote.ts`
- **Actual:** `parseMigrationReport()` only recognizes integer confidence values, so a critical hypothesis reported as `79.9%` is not treated as below the 80% threshold and promote passes.
- **Expected:** Critical hypotheses below 80%, including decimal values such as `79.9%`, must block promote per BRD §13.2 criterion #3.
- **Failing test name:** **"B-018: promote must treat a 79.9% critical hypothesis as below the 80% gate"**.

### B-026
- **Severity:** MEDIUM
- **Component:** `src/templates/features.ts`
- **Actual:** `generateFeatures()` interpolates the first-feature answer directly into markdown, so a Q6/Q12 answer containing markdown headers can inject extra top-level sections and break the file schema.
- **Expected:** Generated `FEATURES.md` must preserve the BRD §9.3 schema regardless of intake text; user input must not create extra top-level sections.
- **Failing test name:** **"B-019: FEATURES template should escape markdown header injection from the first feature answer"**.

### B-027
- **Severity:** MEDIUM
- **Component:** `src/commands/resolve.ts`
- **Actual:** `resolve` marks answered questions as `[x]` in place but leaves them inside `OPEN QUESTIONS`, then adds an empty resolved section instead of moving the item.
- **Expected:** Resolved items must be moved into the resolved section during the resolve lifecycle per BRD Appendix B ("move xuong 'Da tra loi'").
- **Failing test name:** **"B-020: resolve should move answered items into the resolved section instead of leaving them in OPEN QUESTIONS"**.
